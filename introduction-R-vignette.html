Introduction à R

Le langage R

R est un langage de programmation open source multiplateforme spécialisé dans le traitement et l’analyse de données. Son classement en avril 2020, comme le dixième langage le plus populaire au monde, selon l’index TIOBE, traduit son importance comme outil de traitement des données et son développement très actif.

R est une implémentation du langage de programmation S spécialisé dans les statistiques, créé par John Chambers en 1976, combiné au langage Scheme. Il existe des différences importantes entre R et S, mais une grande partie du code écrit pour S s’exécute sans modification dans R. Le langage est développé à partir de 1992 par Ross Ihaka et Robert Gentleman à l’Université d’Auckland. La première version officielle est publiée en 2000.

Interface

Le langage R possède une interface (GUI) limitée qui n’a plus vocation à être directement utilisée. L’utilisateur interagit avec R à travers l’environnement de développement R Studio.

Evolutivité

Les capacités de R sont étendues grâce à des packages créés par les utilisateurs. Un ensemble de packages de base est inclus avec l’installation de R et plus de 15500 packages supplémentaires (en date d’avril 2020 ) sont disponibles sur le dépôt CRAN.

Installer et mettre à jour des bibliothèques (packages).

R fait appel à de nombreux packages qui viennent se greffer sur son noyau (R-Base). Les packages sont téléchargés depuis le site du CRAN par le biais de la fonction install.packages() ou le panneau dédié de R Studio.

Panneau packages dans RStudio.

⚠️ Il est important de distinguer l’installation des packages et leur utilisation à travers la fonction library(). l’installation télécharge les packages sur internet et les installe localement sur le disque dur de l’ordinateur dans le dossier R (Cette opération n’est à réaliser qu’une seule fois). la fonction library() permet de faire appel aux fonctions du package. Elle doit être appelée à chaque fois que cela est nécessaire.

⚠️ Lors d’une nouvelle installation de R, vous serez amené à télécharger un grand nombre de packages.

⚠️ Lors de l’installation des packages par le panneau Packages (ou la fonction install.packages) sous Windows et MacOs, le message There are binary versions available but the source versions are later peut aparaître suivi de la question Do you want to install from sources the package which needs compilation? Cela signifie que le package a été mis à jour récemment sur CRAN mais que le binaire n’est pas encore disponible pour votre système d’exploitation. En effet, certains packages R ont des parties importantes écrites dans d’autres langages compilés, généralement C / C ++ ou Fortran. Ces langages ont besoin d’outils logiciels accessoires pour traduire (“compiler”) leur code source en langage machine qui peut s’exécuter sur un système particulier.
Pour Windows et MacOs, l’auteur d’un package met à disposition des binaires précompilés et ils peut avoir un délais entre leur mise à disposition pour MacOs et Windows. Ainsi, vous ne pouvez que répondre non à la question posée et n’obtiendrez pas la version la plus récente. Vous devrez patienter afin d’attendre une mise en ligne, sauf à forcer et l’installer à partir de la source ( ensemble de dépendances logicielles appelées " RTools ").

Organiser ses fichiers dans un projet

Une session de R pointe systématiquement vers un répertoire définit par l’utilisateur comme répertoire de travail (working directory) pour y lire ou écrire des données stockées dans différents fichiers. En pratique, on associe tous les fichiers relatifs à un même ojbet de travail à un répertoire dédié. RStudio facilite cette gestion avec les “projets”.

  • Créer un projet :
    • Etape 1 : ouvrez RStudio
    • Etape 2 : menu project > New project > Directory name : nom du répertoire > > as subdirectory of U:
  • Ouvrir et fermer un projet :
    • Ouverture depuis le gestionnaire de fichier du système d’exploitation (ex : Windows) : cliquez sur le fichier .Rproj.
    • Fermeture (RStudio : Menu file > quit session). ⚠️ L’option Save workspace permet de sauvegarder les objets présents dans l’environnement.

Console et scripts

Un ordinateur assure des opérations de traitement de l’information qu’il exécute successivement. Ce traitement est mené à l’aide d’instructions.Un programme informatique est constitué d’une suite d’instructions dont l’assemblage dans un ordre précis, R exécutant les instructions de manière séquentielle, conduit au résultat attendu.

On utilise la console lorsqu’il s’agit de saisir des instructions isolées et les scripts lorsque l’on écrit un programme.

Comment saisir une instruction dans la console ?

  • Une instruction est saisie dans la console à droite de l’invite (chevron >).
  • Elle est évaluée par la touche Entrée.
  • Vous pouvez utiliser les flèches haut et bas pour naviguer dans l’historique des commandes que vous avez tapées précédemment.
  • Lorsqu’une instruction est incomplète la console affiche le signe + signifiant que R “attend la suite” :
    • On peut alors compléter l’instruction sur cette nouvelle ligne et valider par Entrée.
    • Tout annuler et revenir à l’invite de commandes normal en appuyant sur la touche Échap.

Comment saisir une instruction dans un script ?

  • On crée un script par le menu File > New file > R Script.
  • On peut exécuter une séquence d’instruction en la sélectionnant et on valide par Run ou Ctrl + Entrée.
  • On peut exécuter la totalité du script par Source (Ctrl + maj + s) ou Source with Echo (Ctrl + maj + Entrée) qui exécute le programme comme si vous aviez copié chaque ligne dans la console.

Quelques bases du langage

Des instructions simples : les expressions arithmétiques

50 + 80 + 200 
## [1] 330
(50 + 80 + 200)/3 
## [1] 110
  • R utilise le point comme séparateur décimal.
  • R utilise les signes opératoires +, -, *, / et ^ et les parenthèses.

Utiliser les variables

  • Les variables permettent de stocker des informations dans la mémoire de l’ordinateur.
  • Pour employer une image, une variable est une boîte, que l’ordinateur va repérer par une étiquette. Pour avoir accès au contenu de la boîte, il suffit de la désigner par son étiquette.
  • Quand on créé une variable “vide” on parle de déclaration en spécifiant son nom.
  • Quand on attribue une valeur à une variable, on parle d’affectation.
  • L’affectation se lit de droite à gauche.
  • Les instructions situées à droite du signe flèche sont affectées à une variable désignée par un nom.
  • La flèche est saisie par le raccourcis clavier Alt-6.
# Exemple de déclaration
var1 <-c()
# Exemples d'affectations
var2 <- 4192
var3 <- 50 + 80 + 200 
var4 <- "Jean Simon"

Afficher le contenu d’une variable

print(var3)     # Affichage du contenu de la variable exp_fr.
## [1] 330
var3            # Ou tout simplement...
## [1] 330

Supprimer une variable

rm(var4)

Réaffecter une valeur à une variable

Il est permis de ré-affecter une nouvelle valeur à une même variable.

var2 <- 4192
var2 <- 6000
var2
## [1] 6000
  • Une variable possède à un moment donné une valeur et une seule.
  • Quand on assigne une nouvelle valeur à une variable déjà existante, la valeur précédente est perdue.

Programme permettant d’échanger les valeurs de deux variables x et y, et ce quel que soit leur contenu préalable.

Utiliser les fonctions existantes

Une fonction est programmée pour accomplir une tâche et retourner un résultat. Les composantes d’une fonction sont son nom, c’est ce nom qui nous permettra de nous y référer et un ou plusieurs arguments, informations nécessaires à la fonction (entre parenthèses et séparés par le signe virgule).

A titre d’exemple, la fonction samplepermet de tirer un échantillon au sein d’un vecteur. Elle prend comme arguments le nom du vecteur ainsi que la taille de l’échantillon.Certains arguments possèdent des valeurs par défaut et n’ont besoins d’être précisés uniquement si l’on souhait être plus explicite ou si on cherche à modifier cette valeur. Ainsi, l’instruction sample(chrono,1, replace = FALSE) equivant à sample(chrono,1).

sample(chrono,1)
## [1] 1673
sample(chrono,3)
## [1] 1564 1541 1536
sample(chrono,1, replace = TRUE)
## [1] 1717
sample(chrono,1, replace = FALSE)
## [1] 1750

Imbrication

Il est possible d’imbriquer des fonctions dans une instruction : fonction1(fonction2())

sort(toupper(ports_bretons))
## [1] "BREST"      "NANTES"     "SAINT-MALO" "VANNES"

Fonctions de base et fonction de packages

  • Il convient de distinguer les fonctions de R {Base} et les fonctions des packages.
  • Une fonction de R {Base} peut ne pas fonctionner au sein d’instructions basées sur des packages.
  • Des fonctions de packages peuvent se substituer à des fonctions de R {Base}.

Règles et conventions de syntaxe

Règles de déclaration

  • Les noms des variables doivent impérativement commencer par une lettre :
1850 <- 1850        # Erreur de déclaration
annee1850 <- 1850        # Syntaxe correcte
  • La casse est significative :
Nantes <- 150
# différent de...
nantes <- 550
  • Le langage lui-même se réserve des termes. Ils sont inutilisables comme noms de variables.
sum <- 250 + 255 + 567    # Erreur de déclaration
somme <- 250 + 255 + 567    # Syntaxe correcte

Conventions de déclaration adoptées en td

  1. Tout en minuscule, séparation par un trait de soulignement (snake_case ) :
ntes_exp_xviie # OK
NtesExpXVIIe # non

  1. Eviter les caractères accentués et les symboles.

  2. Faire court !

jeu_de_donnees
dataset         # A privilégier
france_esclaves_expeditions
fr_esc_exp  # A privilégier

Conventions de style

  • Faire précéder et suivre les opérateurs par un espace :
2 + 2
  • Faire suivre une virgule par un espace :
c(1, 5)
  • Placer un espace avant et après le symbole d’affectation :
a <- 3
  • Ne pas placer d’espace entre une fonction et une parenthèse ouvrante :
sum(2, 5)

Commenter ses instructions

Commenter une instruction

  • Les commentaires sont utilisés pour annoter le code.
  • Les commentaires sont ignorés par Rr.

Exemple

# Commentaire en pleine ligne 
sort(c(4, 1, 3, 2))

sort(c(4, 1, 3, 2))  # Commentaire court en fin de ligne
  • N’excédez 80 caractères par ligne, commentaires inclus pour une question de lisibilité.
  • R ne gère pas les commentaires multilignes
